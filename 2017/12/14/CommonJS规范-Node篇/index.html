CommonJS规范_Node篇

发表于 2017-12-14 | 分类于 前端模块化开发

之前写模块化进化史的时候，提及到为了解决模块化中存在的问题，可以采用模块化规范和项目构建打包。先简要介绍一下模块化规范中的CommonJS规范——Node篇：

0. 前言

Ⅰ. 了解CommonJS规范及其实现

CommonJS规范是最早的JS模块化规范
Node是CommonJS在服务器端的实现
Browserify是CommonJS在浏览器端的实现

Ⅱ. 相关在线资源

commonJS: http://javascript.ruanyifeng.com/nodejs/module.html
NodeJS: http://nodejs.cn/
Browserify: http://browserify.org/

Ⅲ. 相关API

定义暴露模块: exports
    exports.xxx = value
    module.exports = value
引入模块: require
    var module = require('模块名/模块相对路径')

1. 创建项目结构

|-modules
  |-module1.js
  |-module2.js
  |-module3.js
|-app.js
|-package.json
  {
    "name": "commonJS-node",
    "version": "1.0.0"
  }

2. 模块化编码

Ⅰ. module1.js

Ⅳ. app.js

//引用模块
const uniq = require('uniq') // npm install uniq -save
const fs = require('fs')
const module1 = require('./modules/module1')
const module2 = require('./modules/module2')
const module3 = require('./modules/module3')

//使用模块
module1.foo()
module2()
module3.foo()
module3.bar()

console.log(uniq([3, 1, 5, 1, 4, 3]))

fs.readFile('app.js', function (error, data) {
  console.log(data.toString())
})

3. 通过node运行app.js

命令: node app.js
工具: 右键-->运行
