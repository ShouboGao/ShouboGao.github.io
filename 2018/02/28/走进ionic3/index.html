走进ionic3

发表于 2018-02-28 | 分类于 ionic

Ionic

官网：https://ionicframework.com/docs/

安装 cordova ionic

npm install -g cordova ionic

使用 ionic 创建项目

创建浏览器运行的项目：

1.需要安装 nodejs
2.安装 npm install -g cordova ionic
3.创建项目: ionic start myApp tabs
4.cd 到刚才创建的项目
5.ionic serve 运行项目

ionic 创建组件

1.cd 到我们的项目目录
2.ionic g component ComponentsModule
3.ComponentsModule.ts中
    import { TestComponent } from "./test/test";
    @NgModule({
          declarations: [TestComponent],
         imports: [],
         exports: [TestComponent]
    })
4.app.module.ts中引入注册
    import { ComponentsModule } from "../components/components.module";
    @NgModule({    
        ....        
        imports: [BrowserModule, ComponentsModule, IonicModule.forRoot(MyApp)],        
        ....
    })
5.在 pages 里面的页面里面使用这些组件。
    <ion-content padding>
        <test></test>
    </ion-content>

创建页面以及页面跳转

1.ionic g page news
2.xxx.html中 
    <button ion-button color="secondary" (click)='pushButton()'>
3.xxx.ts中
    import { NewsPage } from "../news/news";
    pushButton() {

      }

新增tabs页面

1.创建页面

ionic g page news

2.在 app.module.ts 引入、注册

import { NewsPage } from "../pages/news/news";
declarations: [//声明组件
    MyApp, 
    AboutPage, 
    ContactPage, 
    HomePage, 
    TabsPage,
    NewsPage
],
    entryComponents: [//配置不会在模板中使用的组件
    MyApp,
    AboutPage,
    ContactPage,
    HomePage,
    TabsPage,
    NewsPage
  ],

3.在 tabs.ts 页面引入组件， 配置组件

import { NewsPage } from "../news/news";
export class TabsPage {
  tab1Root = HomePage;
  tab2Root = AboutPage;
  tab3Root = ContactPage;
  tab4Root = NewsPage;
  constructor() {}
}

4.在 tabs.html 配置底部按钮以及图标

<ion-tabs>
  <ion-tab [root]="tab1Root" tabTitle="Home" tabIcon="home"></ion-tab>
  <ion-tab [root]="tab2Root" tabTitle="About" tabIcon="information-circle"></ion-tab>
  <ion-tab [root]="tab3Root" tabTitle="Contact" tabIcon="contacts"></ion-tab>
  <ion-tab [root]="tab4Root" tabTitle="新闻" tabIcon="apps"></ion-tab>
</ion-tabs>

去掉二级页面tabs菜单，修改返回按钮

找到 app.module.ts 修改下面代码

imports: [
    BrowserModule,
    ComponentsModule,
    // IonicModule.forRoot(MyApp)
    IonicModule.forRoot(MyApp, {
      tabsHideOnSubPages: "true", //隐藏全部子页面 tabs
      backButtonText: "返回" //配置返回按钮
    })
  ],

ionic3.x 中的路由导航 NavController 以及路由传值

1.NavController 路由跳转（不传参）

a.需要引入 NavController 模块
    import { NavController } from "ionic-angular";
b.声明 NavController
    constructor(public navCtrl: NavController) {}
c.引入页面跳转
    import { NewsPage } from "../news/news";
    ====分割线====
    pushButton() {
        this.navCtrl.push(NewsPage);
    }
d.在XXX.ts完成上述操作后，再在XXX.html中：
    <button ion-button color="light" (click)='pushButton()'>
        执行button
    </button>

2.NavController 路由跳转（传参）

a.需要引入 NavController 模块(news.ts)
    import { NavController } from 'ionic-angular';
b.申明 NavController （实例化）
    constructor(public navCtrl: NavController) {
    }
c.引入页面跳转传参
    import { NewsinfoPage } from "../newsinfo/newsinfo";
    ====分割线====
    goInfo() {
        this.navCtrl.push(NewsinfoPage, {
          id: "24",
          name: "Kobe"
        });
    }
d.接收页面传参(newsinfo.ts)
    引入 NavParmas 模块
        import { NavParams } from "ionic-angular";
    获取传值
        id: string;
        name: string;
        constructor(public navCtrl: NavController, public navParams: NavParams) {
            this.id = navParams.get("id");
            this.name = navParams.get("name");
        }
e.使用页面参数(newsinfo.html)
    <ion-content padding>
        <h3>{{id}}——{{name}}</h3>
    </ion-content>

3.通过属性路由跳转（传参）

a.引入页面定义数据(news.ts)
    import { NewsinfoPage } from "../newsinfo/newsinfo";
    ====分割线====
    params: any;
    pushPage: any;
    constructor(public navCtrl: NavController, public navParams: NavParams) {
        this.pushPage = NewsinfoPage;
        this.params = { id: 30 };
    }
b.属性绑定跳转的页面(news.html)
    <button ion-button [navPush]="pushPage" [navParams]="params">Go</button>
c.获取传过来的参数(newsinfo.ts)
    id;
    constructor(private navParams: NavParams) {
        this.id = navParams.get("id");
    }
d.使用页面参数(newsinfo.html)
    <ion-content padding>
        <h3>{{id}}</h3>
    </ion-content>

Ionic中 get 请求数据(可参考angular中数据的请求)

1.在 app.module.ts 引入 HttpModule、JsonpModule

import { HttpModule, JsonpModule } from '@angular/http';

2.在 app.module.ts 依赖注入 HttpModule、JsonpModule

imports: [
    BrowserModule,
    ComponentsModule,
    HttpModule,
    JsonpModule,
    // IonicModule.forRoot(MyApp)
    IonicModule.forRoot(MyApp, {
      tabsHideOnSubPages: "true", //隐藏全部子页面 tabs
      backButtonText: "返回" //配置返回按钮
    })
],

3.在需要请求数据的地方引入Http (eg. home.ts)

import {Http,Jsonp} from "@angular/http";

4.构造函数内声明：

constructor(private http:Http,private jsonp:Jsonp) { }

5.对应的方法内使用http请求数据

requestData() {
    // alert('数据请求成功！')
    let _that = this;
    let url = "http://www.phonegap100.com/appapi.php?a=getPortalList&catid=20&page=1";
    this.http.get(url).subscribe(
        function(data) {
            console.log(JSON.parse(data["_body"]));
            _that.list = JSON.parse(data["_body"]).result;
        },
        function(err) {
            console.log(err);
        }
    );
}

##ionic3.x中ion-infinite-scroll上拉分页加载更多

1.Ionic在上拉加载的页面中加入ion-infinite-scroll

<ion-list>
    <ion-item *ngFor="let i of items">{{i}}</ion-item>
</ion-list>
<ion-infinite-scroll (ionInfinite)="doInfinite($event)">
    <ion-infinite-scroll-content loadingSpinner="bubbles" loadingText="Loading more data...">
    </ion-infinite-scroll-content>
</ion-infinite-scroll>

ion-infinite-scroll 的属性：

threshold 距离底部的距离;  
enable 是否何用;
position:button 显示的位置 可选 top

ion-infinite-scroll-content的属性 

loadingSpinner 配置上拉加载更多的图标  
loadingText 上拉加载显示的文字

2.对应页面的业务逻辑中写上拉更新的方法

items = [];
constructor(public navCtrl: NavController) {
    for (let i = 0; i < 30; i++) {
      this.items.push(this.items.length);
    }
}
doInfinite(infiniteScroll) {
    //请求数据
  console.log("Begin async operation");
  setTimeout(() => {
    for (let i = 0; i < 30; i++) {
      this.items.push(this.items.length);
    }
    console.log("Async operation has ended");
    infiniteScroll.complete();//当数据请求完成调用
  }, 500);
}

重要的方法

infiniteScroll.complete(); //当数据请求完成调用
if (this.items.length > 90) {
    infiniteScroll.enable(false); //没有数据的时候隐藏 ion-infinate-scroll
}

##Ionic3.x 中的 ion-refresher 下拉更新
1.Ionic 在下拉加载的页面中加入ion-refresher

<ion-refresher (ionRefresh)="doRefresh($event)">
    <ion-refresher-content 
        pullingIcon="arrow-dropdown" 
        pullingText="Pull to refresh" 
        refreshingSpinner="circles" 
        refreshingText="Refreshing...">
    </ion-refresher-content>
</ion-refresher>

ion-refresher 的重要属性：

closeDuration：关闭 refresher 需要多少毫秒。 默认值是 280。
enabled：是否启用。
pullMax：默认 =pullMin+60 拖至刷新状态的最大距离将自动进入刷新状态。 默认情况下，拉力最大将 pullmin + 60 自动刷新。
pullMin：默认=60 用户必须拉下的最小距离， 直到刷新者进入刷新状态为止。 默认值是60。
snapbackDuration：刷新者需要多少毫秒才能恢复到刷新状态。 默认值是 280。

ion-refresher-content 的重要属性：

pullingIcon：下拉的图标；
pullingText：下拉加载显示的文字；
refreshingSpinner：释放的时候的加载图标；
refreshingText：释放的时候的文字；

2.对应页面的业务逻辑中写下拉更新的方法

doRefresh(refresher) {
    console.log("Begin async operation", refresher);
    setTimeout(() => {
        console.log("Async operation has ended");
        refresher.complete();
    }, 2000);
}

重要的方法

refresher.complete(); //当数据请求完成调用
